# Улучшенный интерфейс просмотра скриншотов игроков (дашборд)

Дата: 2026-07-03
Статус: утверждено пользователем

## Цель

Переработать вкладку «Скриншоты» в разделе «Античит» админ-дашборда: галерея с
миниатюрами, полноэкранный просмотрщик с зумом и листанием, фильтры, массовый
запрос и запрос скриншота из вкладки «Детекты».

**Бэкенд не меняется.** Используются существующие эндпоинты:

- `GET /api/admin/anticheat/screenshots?limit=N` (максимум 500)
- `GET /api/admin/anticheat/screenshots/:id/image` (полноразмерный JPEG, требует Bearer)
- `POST /api/admin/anticheat/screenshots` (`{ nonce }`)
- `GET /api/admin/anticheat/sessions/online`
- `POST /api/admin/anticheat/bans/account` (для кнопки «Забанить» в просмотрщике)

Ретеншн скриншотов на бэкенде — 30 дней, поэтому limit=500 покрывает всю историю.

## 1. Структура вкладки «Скриншоты»

Три зоны сверху вниз:

1. **Онлайн-игроки** — существующий блок, плюс кнопка **«Скриншот у всех»**:
   последовательные POST по каждому nonce; итоговый тост «Запрошено N скриншотов»
   (при частичных ошибках — «Запрошено N из M»). Кнопка заблокирована, пока идёт серия.
2. **Панель фильтров и вида** (одна строка):
   - поиск по логину (подстрока, без учёта регистра);
   - чипы статуса: Все / Готовые (`done`) / В процессе (`pending`+`capturing`) / Ошибки (`failed`);
   - пресет даты: Всё / Сегодня / 7 дней / 30 дней;
   - счётчик «показано N из M»;
   - справа — переключатель вида **галерея ⇄ таблица**, выбор хранится в
     `localStorage` (ключ `anticheat.screenshots.view`).
   Все фильтры клиентские, комбинируются по И. Лимит загрузки списка в
   `anticheat/page.tsx` поднимается со 100 до 500.
3. **Список** — галерея или таблица (см. ниже).

## 2. Галерея и таблица

### Галерея

- Адаптивная сетка карточек: 2 колонки на узком экране, до 5 на широком; превью 16:9.
- Миниатюра = полноразмерный JPEG, загружаемый лениво:
  - IntersectionObserver — грузим только карточки, попавшие во вьюпорт;
  - авторизованный `fetch` с Bearer → `URL.createObjectURL(blob)`;
  - кэш `Map<id, objectURL>` живёт на уровне вкладки (не перезагружаем при
    смене фильтров/вида), все URL revoke при размонтировании вкладки.
- Карточка `pending`/`capturing` — спиннер + подпись статуса (живой прогресс за
  счёт существующего поллинга каждые 2 с при наличии незавершённых).
- Карточка `failed` — иконка ошибки + причина (`error`).
- Подпись карточки: логин, бейдж статуса, дата, кто запросил.
- Клик по `done`-карточке — открытие просмотрщика.

### Таблица

Текущая таблица, плюс первая колонка с мини-превью (та же ленивая загрузка и
общий кэш). Клик по превью/кнопке «Посмотреть» — просмотрщик.

## 3. Полноэкранный просмотрщик (лайтбокс)

Заменяет текущую модалку. Тёмный оверлей на весь экран (`fixed inset-0`).

- **Зум/пан**: колесо мыши — зум к точке курсора; drag — панорамирование;
  двойной клик — переключение fit ⇄ 100%; кнопки «+ / − / сброс».
  Реализация на CSS transform, без новых npm-зависимостей.
- **Листание**: клавиши ←/→ и кнопки-стрелки по краям; навигация по текущему
  **отфильтрованному** списку `done`-скриншотов; счётчик «3 / 17»;
  прелоад соседних изображений; Esc — закрыть.
- **Тулбар**: логин, дата, разрешение, размер файла; кнопки:
  - **Скачать** — blob → `a[download]`, имя файла `login_YYYY-MM-DD_HH-mm.jpg`;
  - **Ещё скриншот** — видна, только если игрок сейчас в online-списке
    (сопоставление по `userUuid`); по клику — новый запрос по его nonce;
  - **Детекты игрока** — закрывает просмотрщик и переключает на вкладку
    «Детекты» с префильтром по логину (в `DetectionsTab` добавляется проп
    начального фильтра, в `page.tsx` — механизм перехода);
  - **Забанить** — confirm-диалог → существующий `POST /bans/account`
    (тот же флоу, что в детектах).

## 4. Живой статус запроса

Поллинг каждые 2 с при наличии незавершённых остаётся. Добавляется: после
**одиночного** запроса (из блока онлайн, из просмотрщика или из детектов)
запоминается id ожидаемого скриншота; когда он переходит в `done`, просмотрщик
открывается автоматически. При массовом запросе («у всех») автооткрытия нет —
карточки дозревают в галерее.

## 5. Запрос из вкладки «Детекты»

- Список онлайн-сессий поднимается из `ScreenshotsTab` на уровень
  `anticheat/page.tsx` и передаётся пропсами в обе вкладки.
- В строке детекта — кнопка-иконка камеры, активная только если `userUuid`
  детекта есть в online-списке. По клику — POST запроса + тост
  («Запрос скриншота отправлен: login»). Автопереключения вкладки нет.

## 6. Структура кода

`dashboard/components/anticheat/`:

- `screenshots-tab.tsx` — оркестрация: онлайн-блок, фильтры, переключатель вида,
  выбор просматриваемого, автооткрытие;
- `screenshot-gallery.tsx` — сетка карточек и табличный вид;
- `screenshot-viewer.tsx` — лайтбокс (зум/пан/листание/тулбар);
- `use-screenshot-image.ts` — хук ленивой авторизованной загрузки изображения
  с общим кэшем objectURL.

Изменяемые существующие файлы: `app/anticheat/page.tsx` (limit 500, подъём
online-списка, переход на детекты с фильтром), `detections-tab.tsx` (кнопка
камеры, проп начального фильтра).

Типы (`Screenshot`, `OnlineSession`) уже есть, бэкенд и типы не меняются.

## 7. Обработка ошибок

- Ошибка загрузки изображения (миниатюра/просмотрщик) — плейсхолдер с иконкой
  и тост один раз, не спамить на каждую карточку.
- Ошибки массового запроса — продолжаем серию, в конце суммарный тост.
- objectURL всегда revoke: при замене, закрытии просмотрщика и размонтировании.

## 8. Проверка

- `cd dashboard && npm run build` — тайпчек (eslint не настроен).
- Визуальная проверка через dev-стек (`./dev.sh`): запрос скриншота, галерея,
  фильтры, зум/листание, скачивание, массовый запрос, кнопка из детектов.
