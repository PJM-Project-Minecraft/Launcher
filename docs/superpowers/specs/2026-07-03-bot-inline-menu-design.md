# Редизайн Telegram-бота: гибридное inline-меню

Дата: 2026-07-03
Статус: утверждён

## Цель

Современный вид бота: вместо reply-клавиатуры на пол-экрана — одно «живое»
меню-сообщение с inline-кнопками, которое бот редактирует на месте
(`editMessageText`). Чат не засоряется, ощущение мини-приложения.

## Принятые решения

- **Направление:** inline-меню (не Mini App, не полировка reply-клавиатуры).
- **Глубина:** гибрид — навигация и экраны inline; диалоги с текстовым вводом
  (пароль, OTP, email, логин/регистрация) остаются текстовыми, но запускаются
  с inline-кнопок и по завершении возвращают в меню.
- **Reply-клавиатура:** одна кнопка `[🏠 Меню]` (persistent, стиль primary),
  короткий placeholder.
- **Шапка:** баннер-картинка через `link_preview_options`
  (`prefer_large_media: true`, `show_above_text: true`) — полный лимит текста
  4096 символов и обычное редактирование сохраняются. Кандидат на баннер —
  `pjm_discord.png` из корня репо.
- **Админка:** остаётся текстовой как сейчас, вход — с inline-кнопки
  (видна только админам).

## Карта экранов

Все экраны — состояния одного меню-сообщения. Везде, кроме главного, есть
кнопка `[← Назад]` (возврат на главный экран).

### Главный экран (аккаунт привязан)

```
[баннер]
🎮 <BrandPublicName>
👋 Привет, <login>!
👤 <login> · 🛡 2FA ✅/❌ · роль

[👤 Профиль]  [🔑 Пароль]
[📧 Email]    [🛡 2FA]
[💎 Донат]    [⬇ Лаунчер]
[🛠 Админка]            ← только при resolveAdmin != nil
```

### Главный экран (не привязан)

Баннер + приветствие + краткая инструкция.

```
[🔑 Войти]  [📋 Регистрация]
[💎 Донат]  [⬇ Лаунчер]
```

### Экраны-состояния

- **Профиль** — карточка: UUID (в `<code>`), логин, маскированная почта
  (`maskEmailUnsafe`), роль, дата регистрации, статус 2FA. `[← Назад]`.
- **2FA** — статус + контекстная кнопка `[Включить]` / `[Выключить]`
  (запускают существующий totp-flow), `[← Назад]`.
- **Донат** — описание + **URL-кнопка** прямо на магазин (`DonateShopURL`),
  `[← Назад]`.
- **Лаунчер** — URL-кнопка «Скачать с сайта» (`LauncherDirectDownloadURL`,
  если задана) + `[📩 Прислать файл в чат]` (текущий `replyLauncherDownload`
  через `sendDocument`), `[← Назад]`.

### Диалоги с вводом

Пароль, email, 2FA-подтверждение, вход, регистрация: callback-кнопка
запускает существующий flow (`repo.SaveDialogue`, состояния `FlowChangePwdOld`
и т.д.) — бот присылает текстовый промпт, дальнейший ввод как сейчас.
По завершении (успех или /cancel) бот **удаляет старое меню-сообщение**
и присылает свежее меню внизу чата.

### Совместимость

Текстовые команды (`/start`, `/profile`, `/donate`, `/launcher`, `/2fa`,
`/password`, `/email`, `/help`, `/cancel`, `/admin`) продолжают работать —
присылают свежее меню, открытое на соответствующем экране. Старые текстовые
лейблы кнопок («Профиль», «Донат»…) в `idleActions` сохраняются на переходный
период.

## Техническая часть

### Callback-роутинг

- `bot.Handle(tele.OnCallback, ...)` в `lifecycle.go`.
- Диспетчер по `callback_data`: `m:home`, `m:profile`, `m:pwd`, `m:email`,
  `m:2fa`, `m:2fa:on`, `m:2fa:off`, `m:donate`, `m:launcher`,
  `m:launcher:file`, `m:login`, `m:register`, `m:admin`.
- Каждое нажатие: `answerCallbackQuery` (снять «часики»), затем
  `editMessageText` или запуск flow.

### Новые хелперы в `telegram/raw.go`

- `EditMessageTextHTML(client, token, chatID, messageID, html, replyMarkup)` —
  raw-вызов `editMessageText` в стиле существующих хелперов.
- `AnswerCallbackQuery(client, token, callbackID, text, showAlert)`.
- Поддержка inline-клавиатур: структура для `inline_keyboard`
  (text + `callback_data` | `url`).
- В `sendMessage`/`editMessageText` — поле
  `link_preview_options: {url, prefer_large_media: true, show_above_text: true}`.

### Баннер

- Статический публичный роут бэкенда, например `GET /api/public/bot-banner.png`
  (файл из `backend/data/`).
- Конфиг бота: `BOT_BANNER_URL`. Пусто — меню работает без картинки
  (link_preview_options не передаётся).

### Отслеживание меню-сообщения

- Запоминаем `message_id` последнего меню на чат: новое поле
  `MenuMessageID` в модели диалога (GORM AutoMigrate добавит колонку) —
  чтобы редактировать его и удалять устаревшее при пересоздании меню.
  Payload для этого не используем: он очищается при `ClearDialogue`.

### Reply-клавиатура

- Везде, где сейчас собирается `mainKeyboardMarkup`, — одна кнопка
  `[🏠 Меню]` (primary). Нажатие = `/start` (свежий главный экран).

### Ошибки

- `editMessageText` упал (сообщение старое/удалено/не изменилось) → шлём
  новое меню-сообщение, старый `message_id` забываем.
- Callback на приватный экран от не-привязанного пользователя →
  `answerCallbackQuery` с alert «Сначала привяжите аккаунт» + главный экран
  в состоянии «не привязан».
- Callback от чужого/протухшего меню (состояние сбилось) → пересоздать меню.

## Тесты

- Юнит на диспетчер callback-данных (маршрут → обработчик).
- Юнит на сборку inline-клавиатур по состоянию: привязан / не привязан /
  админ; наличие/отсутствие URL-кнопок при пустых `DonateShopURL` и
  `LauncherDirectDownloadURL`.
- Стиль — как существующий `admin_test.go`.
- Ручная проверка через тестового бота (отдельный токен).
